# Contributing to Enterprise Dashboard

Vielen Dank für dein Interesse am Enterprise Dashboard! 🎉

## 🚀 Wie kannst du beitragen?

### 🐛 Bugs melden
- Nutze das [Bug Report Template](.github/ISSUE_TEMPLATE/bug_report.md)
- Beschreibe das Problem so detailliert wie möglich
- Füge Screenshots hinzu wenn möglich
- Erwähne dein Betriebssystem und Browser

### 💡 Features vorschlagen
- Nutze das [Feature Request Template](.github/ISSUE_TEMPLATE/feature_request.md)
- Erkläre warum das Feature nützlich wäre
- Beschreibe mögliche Implementierungen

### 🔧 Code beitragen

#### 1. Fork erstellen
```bash
git clone https://github.com/dein-username/enterprise-dashboard.git
```

#### 2. Branch erstellen
```bash
git checkout -b feature/neues-feature
# oder
git checkout -b fix/bug-beschreibung
```

#### 3. Änderungen vornehmen
- Folge dem bestehenden Code-Stil
- Schreibe aussagekräftige Commit-Messages
- Teste deine Änderungen lokal

#### 4. Commit
```bash
git add .
git commit -m "feat: Beschreibung des Features"
```

#### 5. Push und Pull Request
```bash
git push origin feature/neues-feature
```

Erstelle dann einen Pull Request auf GitHub.

## 📝 Commit Message Convention

Wir folgen [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` Neue Features
- `fix:` Bugfixes
- `docs:` Dokumentation
- `style:` Formatierung (keine Code-Änderungen)
- `refactor:` Code-Refactoring
- `test:` Tests
- `chore:` Wartung

Beispiele:
```
feat: Add dark mode toggle
fix: Correct CPU calculation for multi-core systems
docs: Update installation instructions
```

## 🧪 Testing

Stelle sicher dass deine Änderungen nichts kaputt machen:

```bash
npm test
```

## 📋 Checkliste

Vor dem Submit eines PR:
- [ ] Code getestet
- [ ] Keine Lint-Fehler
- [ ] README aktualisiert (falls nötig)
- [ ] CHANGELOG aktualisiert
- [ ] Commit-Messages folgen der Convention

## 🤝 Code of Conduct

- Sei respektvoll und freundlich
- Akzeptiere konstruktive Kritik
- Konzentriere dich auf das Beste für die Community
